# PiliAvalon v1.0.0 Release Notes

## 中文发布摘要

`v1.0.0` 是 PiliAvalon 的第一个正式稳定版。此版本基于 Worksite `main` 的 `5ccc9bf243bab2c5f143032bd2549016a5b857da` 重新验证、重新构建并发布，不复用历史 prebuild APK。

## Changes

- Promotes the accepted Phase 1 shielding work into the first formal stable release after the verified product-main merge.
- Preserves the Phase 1 behavior accepted by the user:
  - recommendation and comment shielding;
  - shielding settings entry;
  - token-match deprecation from normal user-facing rule creation;
  - persisted-rule compatibility and rollback path.
- Publishes freshly built signed APK assets from current Worksite `main`, not the historical prebuild commit.

## Known Risks

- The release asset set is split by ABI: `arm64-v8a`, `armeabi-v7a`, and `x86_64`. There is no universal APK in this release by user-confirmed decision.
- Local Flutter tooling was not used for validation in this environment; GitHub Actions is the recorded release evidence.
- A fresh upstream check on 2026-06-05 found two new PiliPlus commits after the previously checked upstream SHA:
  - `73d7d78080270c374e28bb4d8d1c115086f0addb` (`fix #2295`)

## Rollback Plan

If a release regression is found:

1. Keep `v1.0.0` as the immutable stable release record.
2. Open a narrow hotfix branch from `v1.0.0` source commit `5ccc9bf243bab2c5f143032bd2549016a5b857da`.
3. Build and validate a hotfix release with the same CI, signed build, signing evidence, and runtime-smoke gates.
4. Publish the hotfix as `v1.0.1` after evidence is green.

## User Action Required

Install the APK matching the device ABI:

- `arm64-v8a` for most modern Android phones.
- `armeabi-v7a` for older 32-bit Android devices.
- `x86_64` for emulator or x86_64 Android environments.

Use the release assets attached to `v1.0.0`; do not use the historical `phase-1-prebuild.26799023288` APKs for the formal stable package.
